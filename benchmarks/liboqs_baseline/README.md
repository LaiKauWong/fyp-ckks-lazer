# liboqs KEM Benchmark

This directory contains the benchmark instructions and result files for standard KEMs using the authoritative `liboqs` (Open Quantum Safe) C library.

## Purpose

This benchmark is used as an external reference point for comparing our Lazer-based Module-LWE PKE implementation with existing standardized KEM implementations.

The goal is not to claim superior performance, but to show that the PKE functionality added to Lazer performs within a reasonable and comparable range.

## Benchmark Target

Reference implementation:
- Open Quantum Safe (`liboqs`) C library
- Schemes: `Kyber512`, `FrodoKEM-640-AES`

Measured operations:
- Key generation
- Encapsulation
- Decapsulation

## Files

- `results/liboqs_baseline.txt` — stored benchmark outputs and converted summaries

## Requirements

- C compiler (GCC)
- CMake and OpenSSL development headers
- Internet access on first run to fetch the library

## One-time setup

From the project root:

```bash
cd ~/fyp-ckks-lazer/benchmarks
mkdir -p liboqs_baseline
cd liboqs_baseline
git clone https://github.com/open-quantum-safe/liboqs.git
cd liboqs
mkdir build && cd build
cmake -DOQS_USE_OPENSSL=ON ..
make -j2
```
## Run Benchmark

```Bash
cd ~/fyp-ckks-lazer/benchmarks/liboqs_baseline/liboqs/build
./tests/speed_kem
```

or

```bash
make clean
make bench
```

## Save Output

```Bash
cd ~/fyp-ckks-lazer/benchmarks/liboqs_baseline
mkdir -p results
./liboqs/build/tests/speed_kem > results/liboqs_baseline.txt
```

## Benchmark Configuration

Schemes: Kyber512, FrodoKEM-640-AES

Shared secret size: 256 bits (32 bytes)

Warmup / Benchmark trials: Time-based continuous loop (3 seconds per operation) to ensure heavy CPU warmup and statistical confidence.

## Notes on Fairness

This benchmark measures the core cryptographic operations exposed by liboqs.

To ensure absolute fairness, the raw CPU cycles generated by the tool have been converted to milliseconds (ms) assuming a locked base clock frequency of 2.25 GHz (Time = Cycles / 2,250,000), perfectly matching our PKE benchmark environment.
